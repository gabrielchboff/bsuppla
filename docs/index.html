Statically Analyze Docker Images

bsuppla is a security tool that inspects Docker images without executing them. It scans for supply-chain indicators and suspicious artifacts.

Get Started Learn More

What is bsuppla?

bsuppla (Binary Supply Analysis) is a static analysis tool designed to inspect Docker container images for potential security threats without ever executing the container's contents. It analyzes the image layers, extracts the filesystem, and scans for suspicious artifacts that could indicate supply-chain attacks, backdoors, or compromised base images.

The tool is particularly useful for:

  • Security teams auditing container images before deployment
  • CI/CD pipelines that need to validate base images
  • DevOps engineers wanting to understand what's in their containers
  • Researchers analyzing potentially malicious container images safely

Key Features

Static Analysis Only

Never executes container contents. Inspects image tar and layers completely offline for maximum safety.

Supply-Chain Detection

Identifies suspicious ELF traits, hidden executables, crypto miners, private keys, and risky tools.

Allowlist Support

Suppress known-safe findings with customizable allowlist files. Supports glob patterns.

Baseline Comparison

Compare current findings against a known-good baseline to detect new or missing artifacts.

Suspicion Levels

Outputs an overall suspicion level: low, medium, or high based on detected indicators.

No Dependencies

Runs with just a Rust toolchain and Docker. No external scanners or services required.

Why Static Analysis?

Traditional container security tools often require running containers to analyze them, which introduces significant risks:

  • Zero Execution Risk - Since bsuppla never runs any code from the image, there's no possibility of triggering malicious payloads or establishing reverse shells.
  • Complete Visibility - Can analyze every file in the image including hidden files, configuration, and layer metadata.
  • Fast & Offline - Works without network access. Docker pull/save is the only external operation needed.
  • CI/CD Friendly - Perfect for automated pipelines where executing containers isn't feasible.

Quick Start

Run a Scan 

cd bsuppla/bsuppla
cargo run -- alpine:latest alpine.tar

Example Output

[+] Findings:
  - elf_suspicious: sbin/apk (stripped)
  - startup_script_present: var/spool/cron/crontabs/root (init/rc/cron)
[+] Suspicion level: medium

How It Works

bsuppla follows a multi-stage pipeline to analyze Docker images:

  1. Image Acquisition - Uses Docker to pull and save the image as a tarball (or uses an existing tar).
  2. Layer Extraction - Parses the tar archive and extracts each layer's filesystem.
  3. Binary Analysis - Scans for ELF files, checks permissions, entropy, and applies heuristics.
  4. Signal Detection - Matches findings against known patterns for crypto miners, keys, credentials, etc.
  5. Report Generation - Outputs findings with severity levels and optional baseline comparison.

Use Cases

Image Verification

Before deploying any container image to production, run bsuppla to ensure no suspicious artifacts are present:

bsuppla myregistry.com/myapp:v1.2.3 myapp.tar allowlist.txt

CI/CD Integration

Add bsuppla to your pipeline to block vulnerable images:

# Example: Fail if high suspicion level
bsuppla $IMAGE $IMAGE.tar allowlist.txt
if [ $? -eq 2 ]; then
  echo "High suspicion level detected - blocking deployment"
  exit 1
fi

Baseline Comparisons

Track changes in your base images over time:

# First scan - create baseline
bsuppla alpine:latest alpine.tar --baseline-out baseline.txt

# Subsequent scans - compare against baseline
bsuppla alpine:latest alpine.tar allowlist.txt baseline.txt
