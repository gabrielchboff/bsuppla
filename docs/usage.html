Detection Framework

This document describes the detection framework, allowlist mechanism, and baseline comparison features.

Table of Contents

  • Detection Process
  • Allowlist Mechanism
  • Baseline Comparison
  • Output Format
  • Exit Codes

1. Detection Process

The analysis workflow operates as follows:

1. Image Acquisition
   └─> docker pull & docker save
   
2. Filesystem Extraction
   └─> Parse manifest.json, extract layers
   
3. Traversal
   └─> Walk extracted filesystem tree
   
4. Detection
   └─> For each file: create FileContext, query DetectorRegistry
   
5. Reporting
   └─> Filter, score, compare baseline, output results

2. Allowlist Mechanism

The allowlist feature enables suppression of known-safe findings. This is critical for reducing false positives in operational environments.

2.1 Format

Allowlist entries support two formats:

Path-Only

/path/to/file
/bin/busybox
/usr/lib/libssl.so.3

Kind-Specific

<signal_kind>: <path_pattern>

elf_suspicious: /usr/lib/*
suid_or_sgid_executable: /usr/bin/passwd
risky_tool_present: /usr/bin/curl

2.2 Pattern Matching

Pattern Matches Example
* Any characters /usr/lib/* matches all files in /usr/lib
? Single character /bin/?sh matches /bin/ash, /bin/bsh

2.3 Default Allowlist

If no allowlist is provided, a built-in default is applied to suppress elf_suspicious findings in standard library paths:

/bin/*
/sbin/*
/usr/bin/*
/usr/sbin/*
/lib/*
/usr/lib/*

3. Baseline Comparison

Baseline comparison enables tracking changes in image composition over time.

3.1 Baseline Generation

Generate a baseline from a known-good scan:

bsuppla alpine:latest alpine.tar --baseline-out baseline.txt

3.2 Baseline Comparison

Compare current findings against baseline:

bsuppla alpine:latest alpine.tar allowlist.txt baseline.txt

3.3 Comparison Output

The system reports:

  • New findings: Present in current scan but not in baseline
  • Missing findings: Were in baseline but not found in current scan
  • Baseline matched: Findings present in both

4. Output Format

The system produces structured output indicating findings and suspicion level.

4.1 Example: No Findings

[+] No high-signal findings detected.
[+] Suspicion level: low

4.2 Example: With Findings

[+] Findings:
 - elf_suspicious: sbin/apk (stripped)
 - startup_script_present: var/spool/cron/crontabs/root (init/rc/cron)
[+] Suspicion level: medium

4.3 Example: High Suspicion

[+] Findings:
 - crypto_miner_candidate: usr/bin/xmrig (known miner name)
 - private_key_candidate: root/.ssh/id_rsa (key material present)
 - suid_or_sgid_executable: usr/bin/sudo (mode=4755)
[+] Suspicion level: high

4.4 Example: Baseline Comparison

[+] Findings:
 - crypto_miner_candidate: usr/bin/xmrig (known miner name)
[+] No new findings (all matched baseline).
[+] Baseline matched: 3
[+] Baseline missing:
 - elf_suspicious: old/file.bin
[+] Suspicion level: high

5. Exit Codes

The system returns exit codes indicating scan results:

Code Level Meaning
0 Low No significant findings
1 Medium Findings present, may warrant review
2 High Critical findings detected
