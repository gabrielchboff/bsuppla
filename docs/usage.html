Usage

Learn how to use bsuppla to scan Docker images for security issues.

Basic Usage

Run a simple scan on a Docker image:

bsuppla <image> <image.tar>

Arguments

Argument Required Description
image Yes Docker image name (e.g., alpine:latest)
image.tar Yes Output tar file path
allowlist.txt No Path to allowlist file
baseline.txt No Path to baseline file

Example

bsuppla alpine:latest alpine.tar

Or with cargo run:

cargo run -- alpine:latest alpine.tar

Allowlist File

The allowlist suppresses known-safe findings. Each line should contain a path or pattern. Lines starting with # are comments.

Simple Path

# Suppress findings for common Alpine files
/bin/busybox
/sbin/apk
/usr/lib/libcrypto.so.3

Kind-Specific Allowlist

You can also specify the finding kind to suppress:

<kind>: <path-or-pattern>

Example:

# Suppress all elf_suspicious findings in /usr/lib
elf_suspicious: /usr/lib/*

# Suppress specific private key path
private_key_candidate: /root/.ssh/id_rsa

Pattern Matching

Allowlist entries support glob patterns:

  • * - Matches any characters (e.g., /usr/lib/* matches all files in /usr/lib)
  • ? - Matches a single character (e.g., /bin/?sh matches /bin/ash, /bin/bsh)
Note

If no allowlist is provided, a built-in default allowlist is used for elf_suspicious findings in standard library and binary locations like /lib, /usr/lib, /bin, and /usr/bin.

Baseline File

The baseline file compares current findings against a known-good scan. Each line should match:

<kind>: <path>

Example baseline:

elf_suspicious: /bin/busybox
startup_script_present: /var/spool/cron/crontabs/root

Baseline Comparison Output

When a baseline is provided, bsuppla will:

  • List new findings not in the baseline
  • Show any missing baseline entries (items that were in baseline but not found now)
  • Display unchanged findings (optional, with verbose mode)

Generate Baseline

Export current findings as a baseline file:

bsuppla alpine:latest alpine.tar allowlist.txt --baseline-out baseline.txt

Output

bsuppla prints a list of findings (if any) and an overall suspicion level.

Example Output (Medium)

[+] Findings:
  - elf_suspicious: sbin/apk (stripped)
  - startup_script_present: var/spool/cron/crontabs/root (init/rc/cron)
[+] Suspicion level: medium

Example Output (High)

[+] Findings:
  - crypto_miner_candidate: usr/bin/xmrig (known miner name)
  - private_key_candidate: root/.ssh/id_rsa (key material present)
  - suid_or_sgid_executable: usr/bin/sudo (mode=4755)
  - apk_repository_suspicious: etc/apk/repositories (non-alpine repo: http://example.com/alpine)
[+] Suspicion level: high

Suspicion Levels

Level Exit Code Description
Low 0 No suspicious findings or only minor indicators
Medium 1 Somewhat suspicious findings that may warrant review
High 2 Critical indicators like crypto miners, private keys, or backdoors

Advanced Examples

Offline Scanning

To scan an already-existing tar file without pulling from Docker:

BSUPPLA_SKIP_DOCKER=1 bsuppla alpine:latest alpine.tar

Extract Only (No Scan)

To only extract the image without scanning:

BSUPPLA_SKIP_SCAN=1 bsuppla alpine:latest alpine.tar

Custom Output Directory

To extract to a custom directory:

BSUPPLA_OUTPUT_DIR=/tmp/myfs bsuppla alpine:latest alpine.tar

CI/CD Pipeline Example

#!/bin/bash
IMAGE=$1
TAR_FILE=$(mktemp).tar

# Pull and save image
docker pull $IMAGE
docker save $IMAGE -o $TAR_FILE

# Scan with allowlist
bsuppla $IMAGE $TAR_FILE allowlist.txt
RESULT=$?

# Cleanup
rm -f $TAR_FILE

# Exit based on suspicion level
exit $RESULT
