Detected Signals

bsuppla detects various security indicators in Docker images. This page documents all the signals that can be flagged.

ELF Binary Indicators

These signals relate to executable files and their characteristics.

elf_suspicious

ELF binary with suspicious traits such as being stripped, static, or having packed sections. May indicate obfuscation.

elf_in_writable_dir

Executable found in a writable directory (/tmp, /var/tmp, /dev/shm). Could indicate privilege escalation or runtime injection.

suid_or_sgid_executable

Binary with SUID or SGID bit set, allowing execution with elevated (owner/group) privileges. Common for legitimate tools but risky if unnecessary.

world_writable

File or directory that is world-writable. Attackers can modify these to gain persistence or escalate privileges.

hidden_executable

Executable hidden in a dotfile (starts with .) outside standard locations. Often used to hide malicious code.

elf_outside_standard_paths

ELF binary located outside common bin/lib paths like /bin, /usr/bin, /lib. Unusual placement may indicate compromise.

Credentials & Keys

These signals indicate potentially exposed sensitive data.

private_key_candidate

Private key material found in .ssh/ directory or other sensitive locations. Exposure allows unauthorized access.

ssh_authorized_keys

SSH authorized_keys file present. Could allow passwordless remote access by attackers.

insecure_shadow

/etc/shadow has insecure permissions (world-readable or writable). Password hashes could be extracted.

credential_file

Credential files detected: .env*, .npmrc, .pypirc, .netrc, .aws/*, .kube/config, .docker/config.json. May contain API keys or secrets.

Suspicious Tools

Detection of tools that can be used for both legitimate and malicious purposes.

risky_dual_use_tool

Dual-use tools that can be used for both legitimate and malicious purposes. Includes: curl, wget, nc, socat, ssh, tcpdump, strace, lsof, nmap, netcat, etc.

crypto_miner_candidate

Known cryptocurrency mining software detected (e.g., xmrig, cpuminer, nicehash, ethminer). Indicates the image may be compromised for cryptojacking.

Startup & Persistence

Signals related to persistence mechanisms.

startup_script_present

Startup script detected in init, rc, or cron directories. Could indicate attempts at persistence or scheduled malicious activity.

Package Manager

Detection of suspicious package manager configurations.

apk_repository_suspicious

Suspicious APK repository configuration. Non-standard or unknown repositories could serve compromised packages.

apk_key_suspicious

Unusual APK key name detected. May indicate a malicious package source.

package_manager_suspicious

Suspicious package manager configuration or lockfile source (npm, pip, gem). Could indicate dependency confusion attacks.

Signal Severity

Signals are grouped by potential severity to help prioritize remediation.

High Severity

These findings indicate immediate security risks:

  • crypto_miner_candidate - Active cryptojacking
  • private_key_candidate - Credential exposure
  • insecure_shadow - Password hash exposure
  • credential_file - Secrets exposure

Medium Severity

These findings warrant investigation:

  • suid_or_sgid_executable - Potential privilege escalation
  • startup_script_present - Persistence mechanism
  • risky_dual_use_tool - Tool available for attack use
  • apk_repository_suspicious - Supply-chain risk

Low Severity

These findings may be acceptable depending on context:

  • elf_suspicious - Could be legitimate optimization
  • hidden_executable - May be intentional
  • elf_outside_standard_paths - Could be intentional

Suppressing Findings

Use the allowlist feature to suppress known-safe findings. You can allowlist by path or by signal type:

Path-Based Allowlist

# Suppress all findings for specific paths
/bin/busybox
/sbin/apk
/usr/lib/libcrypto.so.3

Signal-Type Allowlist

# Suppress specific signal types with patterns
elf_suspicious: /usr/lib/*
suid_or_sgid_executable: /usr/bin/passwd
suid_or_sgid_executable: /usr/bin/sudo

Combined Allowlist

# Example allowlist.txt
# Suppress specific paths
/bin/busybox
/sbin/apk

# Suppress signal types with patterns
elf_suspicious: /usr/lib/*
risky_dual_use_tool: /usr/bin/curl
risky_dual_use_tool: /usr/bin/wget
