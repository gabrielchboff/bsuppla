Signal Taxonomy

This document provides a comprehensive taxonomy of security signals detected by the bsuppla framework.

1. Overview

The bsuppla framework detects security-relevant artifacts through a multi-category signal taxonomy. Each signal represents a specific type of finding with associated risk characteristics.

2. ELF Binary Signals

Analysis of ELF (Executable and Linkable Format) binaries provides insight into binary characteristics that may indicate obfuscation or compromise.

Signal Description Analysis Method
elf_suspicious ELF binary with stripped symbols, static linking, or suspicious sections (UPX, packer) ELF header/section parsing
elf_in_unusual_location Executable located outside standard paths (/bin, /usr/bin, /lib) Path prefix matching

2.1 Detection Criteria

  • Stripped Binary: Missing .symtab or .strtab sections
  • Static Binary: No PT_INTERP or PT_DYNAMIC program headers
  • Packed Binary: Section names containing "upx", "packer", "packed"
  • Unusual Location: Path does not start with /bin, /sbin, /usr/bin, /usr/sbin, /lib, /usr/lib

3. Permission Signals

Unix permission analysis identifies potentially dangerous file access configurations.

Signal Description Threshold
suid_or_sgid_executable Executable with SUID (4000) or SGID (2000) bit set mode & 0o6000 != 0
world_writable_executable Executable that is world-writable mode & 0o002 != 0
executable_in_writable_dir Executable located in /tmp, /var/tmp, /dev/shm, /run, /var/run Path prefix matching
hidden_executable Executable with filename starting with '.' Filename starts with '.'
insecure_shadow_permissions /etc/shadow with world-readable or writable permissions Path + permission check

4. Credential Signals

Detection of exposed sensitive data including cryptographic keys and authentication artifacts.

Signal Description Detection Method
private_key_candidate SSH private key files: id_rsa, id_ed25519, id_dsa, id_ecdsa, *.pem in ~/.ssh/ Path pattern + filename matching
authorized_keys_present SSH authorized_keys file at ~/.ssh/authorized_keys Exact path match
credential_file Configuration files containing credentials: .env, .npmrc, .pypirc, .netrc, .aws/, .kube/config, .docker/config.json Filename + path combination

5. Tool Signals

Identification of tools with dual-use potential or known malicious associations.

Signal Known Indicators Risk Category
risky_tool_present curl, wget, nc, ncat, netcat, socat, ssh, scp, sftp, strace, tcpdump Dual-use
crypto_miner_candidate xmrig, xmr-stak, minerd, cgminer, bfgminer, kawpowminer, cryptonight Cryptojacking

6. Persistence Signals

Detection of mechanisms that enable persistent execution across reboots or container restarts.

Signal Path Patterns
startup_script_present /etc/init.d/*, /etc/rc*, /etc/cron*, /var/spool/cron/*, /etc/crontab

7. Supply-Chain Signals

Analysis of package manager configurations that may indicate compromised or malicious package sources.

Signal Configuration Files Anomaly Detection
package_manager_config_suspicious /etc/npmrc, ~/.npmrc, /etc/pip.conf, ~/.pip/pip.conf, /etc/gemrc, ~/.gemrc Non-standard registry URLs
lockfile_suspicious_source package-lock.json, yarn.lock, Pipfile.lock, poetry.lock, Gemfile.lock git+ or http:// sources
apk_repository_suspicious /etc/apk/repositories Non-alpine repositories, file:// URLs
apk_key_unusual /etc/apk/keys/* Non-.rsa.pub key filenames

8. Severity Classification

Each signal is assigned a severity level based on potential impact and exploitability.

Level Score Signals
Critical 5 private_key_candidate, credential_file, crypto_miner_candidate
High 4 suid_or_sgid_executable, world_writable_executable, insecure_shadow_permissions
Medium 3 package_manager_config_suspicious, apk_repository_suspicious, apk_key_unusual
Low 2 elf_suspicious, elf_in_unusual_location, executable_in_writable_dir, startup_script_present

8.1 Suspicion Level Calculation

The overall suspicion level is computed by summing individual signal scores:

total_score = Σ signal_score
suspicion_level = 
  if total_score >= 10: "high"
  else if total_score >= 4: "medium"
  else: "low"
