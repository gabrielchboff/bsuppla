Installation

Get bsuppla up and running on your system. Follow the instructions below based on your setup.

Requirements

Before installing bsuppla, ensure your system meets the following requirements:

Requirement Description Notes
Rust Toolchain Stable Rust (1.70+) Required for building from source
Docker Docker daemon running Required for pulling/saving images
Operating System Linux, macOS, Windows (WSL2) Primary target is Linux
Memory 4GB+ RAM recommended For larger container images

Build from Source

The recommended way to install bsuppla is building from source. This ensures you have the latest version and allows you to inspect the code.

Step 1: Clone the Repository

git clone https://github.com/gabrielchboff/bsuppla.git
cd bsuppla

Step 2: Build the Project

cd bsuppla/bsuppla
cargo build --release

The build process will:

  • Download and compile all Rust dependencies
  • Optimize the binary for release
  • Produce a single static executable

Binary Location

After building, the binary will be located at:

bsuppla/bsuppla/target/release/bsuppla

Install

You can install bsuppla using the provided install script or Make:

Using install script

bash scripts/install.sh

Using Make

make install
Note

The install script copies the binary to /usr/local/bin by default. You may need sudo privileges depending on your system configuration.

Custom Installation Path

To install to a custom directory, set the PREFIX variable:

PREFIX=$HOME/.local bash scripts/install.sh

Verify Installation

Check that bsuppla is installed correctly:

bsuppla --help

You should see output similar to:

bsuppla 1.0.0
Statically analyze Docker images without executing them

USAGE:
    bsuppla <IMAGE> <OUTPUT_TAR> [ALLOWLIST] [BASELINE]

For more information, try --help

Environment Variables

bsuppla supports the following optional environment variables:

Variable Description Default
BSUPPLA_SKIP_DOCKER=1 Skip docker pull/save - useful for scanning an existing tar file false
BSUPPLA_SKIP_SCAN=1 Skip scanning after extraction false
BSUPPLA_OUTPUT_DIR=/path Set extraction directory container_fs

Usage Examples

Scan existing tar

BSUPPLA_SKIP_DOCKER=1 bsuppla alpine:latest alpine.tar

Extract only (no scan)

BSUPPLA_SKIP_SCAN=1 bsuppla alpine:latest alpine.tar

Custom output directory

BSUPPLA_OUTPUT_DIR=/tmp/myfs bsuppla alpine:latest alpine.tar

Troubleshooting

"command not found" after installation

Ensure the installation directory is in your PATH:

export PATH="$PATH:/usr/local/bin"

Docker permission denied

Ensure your user has permission to run Docker commands:

sudo usermod -aG docker $USER

Build errors

Ensure Rust is up to date:

rustup update
cargo update
